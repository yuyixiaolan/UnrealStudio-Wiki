## 新项目注册

启动新项目时，请参照[项目模板](./project-template)创建文档，确保：

- 项目目标清晰可衡量
- 技术选型有依据
- 踩坑记录实时更新
- 结项时附上可运行的 Demo / 可执行文件 / 项目包

## 文档规范

项目文档统一使用 Markdown，存放于 `docs/projects/` 目录。每篇文档建议包含：

```
---
title: 项目名称
author: 作者
date: 开始日期
status: 进行中 / 已结项
---

# 项目名称

## 概述
一句话说明这个项目做什么。

## 技术栈
用到的引擎版本、插件、第三方库。

## 开发日志
按时间倒序记录关键决策、踩坑、解决方案。

## 成果展示
截图、视频链接、Demo 下载地址。
```
